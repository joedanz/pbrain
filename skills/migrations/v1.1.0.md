# v1.1.0 Migration: Tech-Domain Taxonomy

This migration reshapes the brain schema for senior-engineer use cases. It ADDS
seven new directories, re-skins `companies/` for tech-org tracking, and DROPS
seven VC/founder-specific directories that PBrain's audience doesn't use.

**This migration does NOT touch existing content.** It updates the resolver and
templates only. Existing pages keep working; new pages get the new homes.

If you never filed anything under the dropped directories (`deals/`, `hiring/`,
`civic/`, `org/`, `media/`, `personal/`, `household/`, `programs/`), you have
nothing to do except pull the updated skills and RESOLVER.md.

## What Changed

### Directories ADDED (7)

| Directory | For |
|-----------|-----|
| `libraries/` | Software libraries you consume (npm, PyPI, crates.io, etc.) |
| `ai-tools/` | AI products, models, agent runtimes you drive (Claude Code, Cursor, GPT-5…) |
| `repos/` | Git repos you own or closely follow as source trees |
| `patterns/` | Reusable code patterns/idioms (package-independent) |
| `papers/` | Research papers (arxiv, conference proceedings) |
| `talks/` | Conference talks, podcasts, video lectures |
| `books/` | Technical books and long reference works |

Templates for all seven live in `docs/PBRAIN_RECOMMENDED_SCHEMA.md`.

### `companies/` template re-skinned

The company page is now tech-org focused. Old (VC) sections are gone:
- ❌ Stage (Seed / Series A / Growth)
- ❌ Key metrics (revenue, headcount, funding)
- ❌ Investors / board

New sections:
- ✅ What they make
- ✅ Current offerings (models, APIs, CLIs, hosted services)
- ✅ Direction (roadmap signals, recent announcements)
- ✅ My usage (which endpoints, which model, your observed cost)
- ✅ Key people (CEO, research lead, head of DX — not investors)

**Existing company pages are NOT rewritten.** Update them as you touch them;
new pages use the new template.

### Directories DROPPED

The following are no longer part of the recommended PBrain schema:
`deals/`, `hiring/`, `civic/`, `org/`, `media/`, `personal/`, `household/`,
`programs/`.

Existing pages in these directories are left alone. If you want to preserve
them: do nothing. If you want to move them: the resolver will not route new
content there, so they'll slowly become stale archives.

### Enrichment sources rewired

`skills/enrich/SKILL.md` now prioritizes GitHub API, package registries,
arxiv / Semantic Scholar, and product docs over VC-style LinkedIn/Crunchbase
lookups. Tier tables and data-source priority order are updated.

## Agent Instructions

### Step 1: Pull the updated resolver and skills

No-op if you're syncing normally — the changes are documentation + routing only.

### Step 2: Sanity-check the new directories exist

```bash
cd "$(pbrain config brain_path 2>/dev/null || echo ~/brain)"
mkdir -p libraries ai-tools repos patterns papers talks books
```

Only create the ones you'll actually use. If you never track papers, skip
`papers/` — the resolver will file paper content under `inbox/` until you're
ready to opt in.

### Step 3: Seed at least one `libraries/` page (optional)

```bash
pbrain enrich library zod
```

Confirms the enrich skill writes a valid library page with the new template.

### Step 4: Log migration complete

```bash
mkdir -p ~/.pbrain/migrations
echo '{"version":"1.1.0","ts":"'$(date -u +%Y-%m-%dT%H:%M:%SZ)'","status":"complete"}' >> ~/.pbrain/migrations/completed.jsonl
```

## Notes for Users Coming from GBrain

If you're migrating a GBrain v0.10.1 brain to PBrain v1.1.0:
1. Your existing `deals/`, `hiring/`, `civic/` etc. pages are untouched. Keep
   them as archives or move relevant content to other directories.
2. Your existing `companies/` pages keep working — the template change only
   affects NEW pages.
3. Consider re-enriching your top 10 tech-company pages to pick up the new
   "What they make / Current offerings / Direction / My usage" sections.

## Rollback

If this migration causes issues, the only destructive change is directory
routing — no content is modified. To roll back:
1. Restore `docs/PBRAIN_RECOMMENDED_SCHEMA.md` and `skills/RESOLVER.md` to
   their pre-migration state from git history.
2. Remove the completion marker:
   `grep -v '"version":"1.1.0"' ~/.pbrain/migrations/completed.jsonl > /tmp/m && mv /tmp/m ~/.pbrain/migrations/completed.jsonl`

No data loss is possible because this migration only rewires the routing
table.
